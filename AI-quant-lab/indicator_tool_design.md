# 交互式技术指标分析工具 — 产品设计文档

## 1. 产品概述

### 1.1 定位
一个纯前端、单页的 HTML 工具，让用户选择已取好的股票数据，自由调节五大技术指标参数，实时重绘图表。**零后端依赖**，数据预加载自本地 JSON。

### 1.2 核心价值
- 现有 notebook 每次改参数要重新运行，交互感弱
- 本工具将 notebook 的计算逻辑封装为可交互的 UI，参数一调、图表即变
- 适合探索性分析：快速试不同参数组合，观察指标形态变化

### 1.3 技术约束
- 纯 HTML + CSS + JavaScript（单文件 + data.js 数据文件）
- 数据从内嵌的 `data.js` 文件中直接读取（方案A，零网络依赖）
- 双击 `index.html` 即可离线使用，无需 HTTP 服务器
- 图表使用 Chart.js (CDN 加载，首次需要网络)
- 所有计算在浏览器端完成

## 2. 功能需求

### 2.1 数据选择

| 功能 | 描述 |
|------|------|
| 股票选择器 | 下拉框选择：中芯国际(A+H)、比亚迪(A+H)、长江电力(A) |
| 市场选择 | A股 / 港股（仅在该标的有对应市场数据时可用） |
| 复权选择 | 不复权 / 前复权 / 后复权（A股适用） |
| 日期范围 | 可选最近 N 个交易日（全部 / 1月 / 3月 / 6月 / 1年），默认全部 |

### 2.2 指标计算与参数调节

| 指标 | 可调参数 | 默认值 | 参数范围 | 简要说明（悬停/固定显示） |
|------|---------|--------|---------|--------------------------|
| **RSI** | 周期 | 14 | 2~50, 步长1 | 衡量近期涨跌幅力度，>70超买、<30超卖 |
| **MACD** | 快线周期 / 慢线周期 / 信号周期 | 12 / 26 / 9 | 2~60 / 5~100 / 2~30 | 双均线差值识别趋势转折，金叉看涨死叉看跌 |
| **ATR** | 周期 | 14 | 2~50, 步长1 | 衡量真实波动幅度，用于设定止损宽度 |
| **布林带** | 周期 / 标准差倍数 | 20 / 2 | 2~60 / 1~4(0.5步长) | 统计学价格通道，带宽收缩预示变盘 |
| **OBV** | 信号线周期 (0=关闭) | 0 | 0~30, 步长1 | 能量潮，量先于价，验证趋势真伪 |

每个参数提供两种输入方式:
- **滑块 (range slider)**: 快速拖动调节，直观感受变化
- **数字输入框 (number input)**: 精确键入数值，回车确认
- 二者联动：拖动滑块 → 输入框数值同步更新；修改输入框 → 滑块位置同步移动

**控制面板布局**: 位于页面左侧，竖向排列，每个指标占一个卡片区块，
包含指标名称、简要说明（约 8~15 字）、以及参数调节行（滑块 + 输入框并排）。

**关键**: 左侧每个指标卡片的垂直位置与右侧对应图表**严格对齐**，
即 RSI 参数卡 ↔ RSI 图表、MACD 参数卡 ↔ MACD 图表 处于同一行水平位置。
图表行之间留 8~12px 间隙。

### 2.3 图表展示

| 图表 | 类型 | 说明 |
|------|------|------|
| **主图 — 价格** | 折线图 / K 线图（可选） | 背景透明，叠加均线可选 |
| **RSI 图** | 折线图 | 0~100 范围，标记 30/50/70 水平线 |
| **MACD 图** | 折线图 + 柱状图 | DIF/DEA 折线 + 红绿 MACD 柱 |
| **ATR 图** | 面积图 | ATR 曲线 + 填充面积 |
| **布林带图** | 带状图 | 上轨/中轨/下轨 + 价格叠加 |
| **OBV 图** | 折线图 | OBV 累计曲线 + 可选信号线 |

**交互需求**:
- 鼠标 hover 时显示十字准星 + 数据提示 (tooltip)
- 可缩放拖拽（可选 v2 功能）
- 图表联动：缩放一个图，其他同步

### 2.4 UI 布局

核心原则：**左右水平对齐** — 每个指标的参数卡片与对应的图表位于同一行，视觉上形成"左参数 + 右图表"的配对结构。

**关键设计原则**:
- **水平对齐**: 左侧参数卡片和右侧图表共享同一垂直位置（用相同的 y 起始和高度），形成"行"的感觉
- **行间距**: 每个指标行之间保留 8~12px 空隙，避免视觉压迫
- **高度差异处理**: 不同指标的参数行数不同（RSI 1行参数 vs MACD 3行参数），参数卡片高度应与对应图表高度匹配
- **图表统一高度**: 同一行的左右两侧使用相同高度，确保底部平齐

## 4. UI/UX 设计规范

### 4.1 配色方案
- **遵循中国股市惯例**: 涨红色、跌绿色
- 指标线颜色:
  - RSI: 紫色 `#7F77DD`
  - MACD DIF: 蓝色 `#185FA5`, DEA: 橙色 `#D85A30`
  - ATR: 绿色 `#639922`
  - 布林带上轨/下轨: 橙红, 中轨: 绿色
  - OBV: 粉色 `#D4537E`, 信号线: 灰蓝 `#888780`
- 背景: 浅灰 `#F5F5F5`, 卡片: 白色

### 4.2 响应式断点
| 屏幕 | 布局 |
|------|------|
| >1200px | 控制面板左侧固定(宽260px) + 右侧图表区域 |
| 768~1200px | 控制面板折叠为顶部可展开面板 + 图表全宽 |
| <768px | 所有控件堆叠，图表自适应高度 |

### 4.3 交互细节
- 滑块拖动时**去抖防抖** (debounce 300ms)，避免频繁重绘
- 参数输入框支持直接键入数字，回车确认
- 图表 hover 时同步显示所有子图的 tooltip
- 加载/切换数据时显示骨架屏 loading

## 7. 风险与注意事项

1. **浏览器内存**: 全部数据加载到内存，约 4 个 × 250 条 × 11 字段 ≈ 极少量，无风险
2. **Chart.js 性能**: 4 个子图同时渲染，canvas 重绘对现代浏览器无压力
3. **数据一致性**: 切换股票时需清除旧图表状态，避免残留
4. **数据时效**: 内嵌数据为 2026-07 静态快照，如需最新数据需重新生成 `data.js`
5. **移动端适配**: 滑块的触控体验需要优化
